# Plan: Route AskUserQuestion Through cosa-voice MCP Tools

**Date**: 2026-03-02
**Session**: 74
**Status**: Implemented

## Context

Claude Code's built-in `AskUserQuestion` tool renders **only in the terminal UI** ‚Äî no audio alert. Since the user's mental model is "I am NOT watching the terminal," preference questions asked via `AskUserQuestion` get missed entirely. The cosa-voice MCP server provides equivalent blocking tools (`ask_multiple_choice`, `ask_yes_no`, `converse`, `ask_open_ended_batch`) that deliver **both audio alerts AND UI**, ensuring the user is always notified.

Currently there is **zero routing guidance** anywhere ‚Äî no rule says "prefer cosa-voice over AskUserQuestion." The `ask_multiple_choice()` format is already AskUserQuestion-compatible (same `questions` array schema), so this is purely a behavioral directive, not a code change.

## Approach: Both Global CLAUDE.md + Notification Skill

**Why both?**
- **Global CLAUDE.md** is always loaded into context ‚Äî the mandate must live here to be active from the first turn, even before the notification skill triggers
- **Notification skill** provides the detailed routing table as supplementary reference

**Why not skill-only?** Skills load on-demand based on trigger detection. If Claude doesn't detect a notification trigger early in a conversation, it may use `AskUserQuestion` before the skill activates.

## Changes

### 1. Global CLAUDE.md ‚Äî Add routing mandate

**File**: `~/.claude/CLAUDE.md`
**Location**: Inside the existing "CLAUDE CODE NOTIFICATION SYSTEM" section, after the "Available MCP Tools" table

Added a new subsection:

```markdown
### INTERACTIVE TOOL ROUTING (AskUserQuestion ‚Üí cosa-voice)

**MANDATE**: ALWAYS prefer cosa-voice MCP blocking tools over the built-in `AskUserQuestion` tool.

**Routing Table**:

| Scenario | Use This | NOT This |
|----------|----------|----------|
| Binary yes/no decision | `ask_yes_no()` | `AskUserQuestion` with 2 options |
| Multiple choice (2-4 options) | `ask_multiple_choice()` | `AskUserQuestion` |
| Open-ended clarification | `converse()` | `AskUserQuestion` with "Other" |
| Multiple open-ended questions | `ask_open_ended_batch()` | Multiple `AskUserQuestion` calls |
```
