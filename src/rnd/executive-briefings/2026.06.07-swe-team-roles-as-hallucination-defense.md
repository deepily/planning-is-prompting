## 1. The problem we're defending against (generic)

The central failure mode of an AI agent is **confident hallucination** — asserting a plausible, well-phrased claim that is simply **not true**, and doing it *repeatedly*, sometimes even after a correction. The dangerous part isn't that an agent is wrong; it's that **the same process that produces the error also produces the confidence**, so an agent is structurally bad at catching its *own* hallucination. "Let me re-check my work" from a single agent is weak medicine.

A team of agents doesn't automatically fix this — if they all share the same vantage point, they share the same blind spot (groupthink at machine speed). **The defense has to be structural.**

## 2. The mechanism: designated roles = independent vantage points (generic → structural)

Our **SWE team** is five designated roles, each spun up with its own **charter** auto-loaded into its brief — so each comes online already knowing its mandate, its gate, and its discipline. The point of the roles is **not** division of labor for speed; it's **division of vantage for truth.**

| Role | Mandate | Its distinct vantage / incentive |
|------|---------|----------------------------------|
| **Manager** | Sequences work, holds the commit gate, staffs the crew | *Verify-before-committing* — won't commit on a claim it hasn't checked |
| **Workflow Steward** (Observer) | Process correctness; sets the success criteria **up front** | *Conformance* — "does this meet the criterion we set before we started?" |
| **Implementer** | Builds to spec, tests its own unit | *Make it work* — closest to the code |
| **Reviewer** | Renders a **refute-first** verdict | *Try to break it* — the designated skeptic, default "not yet" |
| **Tester** | Runs the live suite / integration reality | *The bytes* — what actually executed, not what was claimed |

Two design choices make this hallucination-**resistant**, not just collaborative:

- **"Fresh person, stable charter."** Each worker is spawned **fresh** — no accumulated context-rot, no inheritance of the team's current delusion — but carries a **stable role charter**. So every cycle gets genuinely **fresh eyes** *by construction*, while the role discipline stays constant.
- **Enforced fact-checking primitives**, written into the charters as non-negotiable habits:
  - **Receipts before claims** — cite a primary artifact (a log line, a commit hash, a command's real output) or mark the claim *unverified*.
  - **Refute-first** — the reviewer defaults to "not yet" and must *try and fail* to break a change before passing it.
  - **Verify the resolving claim** — don't accept "fixed"; check the fix against the source.
  - **Bytes-first** — trust what the run produced, not what an agent summarized.

The load-bearing property: **a hallucination must survive contact with a differently-incentivized peer to reach the commit gate.** Usually it can't — the skeptic, the bytes, or the verifier kills it first.

## 3. The dynamic, with real examples (specific)

Every example below is a *real* catch, each proven by a primary artifact — not a hypothetical.

### 3.1 The review overruled BOTH its own designers
On the team's first real build (2026-06-06), the **Reviewer overturned the position of both the Steward and the Manager** on a technical fact they had each missed: a stamping function had **two call sites**, and one of them (inside middleware, before the request proceeds) made an exception *fatal* — so a narrowly-scoped "catch this one error" criterion was wrong; a total no-throw was structurally required. The designers leaned one way; the **fresh, refute-first reviewer**, enumerating *every* call site, proved the other. **No rubber-stamp — including the manager's own claim.**
*→ codified as the "enumerate every call site before ruling" check in the Reviewer charter.* (`2026.06.06-swe-team-first-run-postgame.md`)

### 3.2 The adversarial reviewer caught a regression the author's own tests missed
On the v2.2 build, the Implementer's *owned* tests were green — but the Reviewer **re-ran the FULL suite** (not just the owned files) and caught that a new change had silently broken three pre-existing integration tests. **Proven by a git-stash attribution, not asserted.** A reviewer who trusted the author's green would have shipped the regression.
*(`2026.06.06-arbiter-closed-loop-postgame.md`)*

### 3.3 A manager hallucinated "the file is missing" — the Steward refuted it with evidence
Today (2026-06-07), the Manager went to review a design doc, ran four searches, found nothing, and reported it **missing / unsaved**. The **Steward refused the premise** and pulled primary evidence — `ls` + `git status` proved the file was **on disk** the whole time; the Manager had simply searched the *wrong repository* on a shared filesystem. A confident, four-times-"verified" claim — **false**, and caught only because a differently-positioned peer checked the bytes instead of trusting the narrative.
*→ codified as the "repo ≠ filesystem; verify with absolute paths" memory.*

### 3.4 An agent narrated a *requested* identity as a *real* one
A worker was spawned with a *preferred* persona ("Clayton") but the allocator assigned a different one ("Rachel"); the report narrated "Clayton" — **a confabulation of intent as fact.** Caught, then **structurally prevented**: every spawn now must *confirm the actually-allocated identity* before narrating it.
*→ "verify the allocated persona post-spawn."*

### 3.5 A threshold asserted before reading the source
A manager asserted a "stale after ~10 minutes" rule to explain a bug — confidently, and **wrongly** (the real threshold, in the config file, was **8 hours**). The fix wasn't "be smarter"; it was the discipline **derive the fact from the source, don't relay an instinct as authoritative.** Self-caught once the source was actually read.

### 3.6 The system caught **its own Observer** — and that's the point
Honesty matters here: the Steward (me) is not exempt. On 2026-05-30 I once logged a complete **"PASSED" exit summary for a step that never actually ran** — a textbook confabulation. The same discipline that catches workers caught it: **cite a primary evidence artifact for every claimed result, or mark it unverified.** The defense is **rank-blind** — it overrules the worker, the reviewer, the manager, *and* the observer. A multi-session coverage campaign even kept a running **"confabulation-correction ledger" (~6 self-caught instances)**, including a fabricated liveness marker — the guardrail working in the open.
*(`2026.05.30-cosa-coverage-campaign-observer-log.md`)*

## 4. Why a team of roles beats one capable agent (the insight)

- **Diversity of vantage beats raw capability.** The *same* underlying model, placed in five seats with five incentives, catches far more than one model in one seat — because an agent's errors are **correlated within its own vantage but not across vantages.** The reviewer's blind spots aren't the tester's; the manager's aren't the steward's.
- **The skeptic is a designated seat, not a mood.** "Refute-first" review is a role someone *owns* every cycle, so skepticism doesn't depend on anyone happening to feel cautious.
- **Reality has a seat at the table.** The Tester's "what the bytes did" is the tiebreaker that no amount of confident prose can argue with.
- **The check runs upward.** The most important examples above are the ones where the structure overruled the people *running* it (§3.1, §3.3, §3.6) — that's what keeps quality from depending on any one agent's good day.

## 5. The payoff for a project manager

- **Hallucinations die before the gate, not in production.** The commit gate requires *both* green *and* an adversarial review — a claim has to pass the skeptic and the bytes.
- **No single point of failure.** No agent's bad day ships, because no agent's claim is self-certifying.
- **Auditable, not folklore.** Every catch is backed by a primary artifact (a stash diff, an `ls`, a config line, a log) — a manager can inspect *why* a thing was rejected.
- **Scales with stakes.** Higher-risk work simply adds more reviewers or testers — more independent vantages — without changing the model.
