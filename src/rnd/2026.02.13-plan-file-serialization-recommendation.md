# Plan File Serialization: Research & Recommendation

**Date**: 2026.02.13
**Status**: Implemented
**Deliverables**: Canonical workflow, global skill, CLAUDE.md directive

---

## Executive Summary

Claude Code's plan file naming convention (`[adjective]-[gerund]-[noun].md`) produces semantically meaningless filenames. After 31 days of usage, 169 plan files have accumulated in `~/.claude/plans/` with zero searchability. This document presents the research data and recommends formalizing an existing ad-hoc serialization pattern.

## Recommendation

Formalize the ad-hoc serialization pattern as a cross-project behavioral directive:

1. **CLAUDE.md section** (~20 lines): Brief mandate with example and references
2. **Global skill** (`~/.claude/skills/plan-serialization/SKILL.md`): Trigger-activated guidance
3. **Canonical workflow** (`workflow/plan-serialization.md`): Full rationale, patterns, decision criteria

### Decision Criteria (Condensed)

```
Serialize if: >1KB AND (architectural OR >30min development OR needs future recall)
Skip if: <1KB OR abandoned OR trivial fix
```

### Naming Format

```
yyyy.mm.dd-descriptive-slug.md  (3-6 hyphenated words capturing the plan's SUBJECT)
```
