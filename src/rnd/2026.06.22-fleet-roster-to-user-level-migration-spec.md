# Migration Spec: Fleet-Roster → User-Level Single Source (`~/.claude/fleet-roster.env`)

**Date**: 2026-06-22 · **Author**: María 🌸 (design) · **Builder**: Mr Radio's crew (lupin infra) · **Reviewer**: Tiberius · **Status**: SPEC — Rick-approved 2026-06-22 (build + hold; systemd restart = Rick's gate)

**Goal**: make the declared-manager roster a SINGLE fleet-wide list naming every repo's manager(s), stored repo-agnostically at `~/.claude/fleet-roster.env` (alongside `settings.json` + the per-project memory dirs), so no product repo owns the fleet roster. Then the subagent-governance hook DERIVES its manager predicate from this one source (+ the preferred-persona head) instead of a separate hand-set `LUPIN_MANAGER_PERSONAS`.

## 3. Test-migration plan (the part that needs care — Rick's catch)

`lupin/src/tests/unit/test_fleet_roster_env.py` (10 tests) is the affected suite. It currently resolves the roster via `LUPIN_ROOT/src/conf/fleet-roster.env` (`:27`) and controls file-presence via `LUPIN_ROOT`. Moving to `~/.claude` breaks the path-coupled tests AND — if naively re-pointed to `$HOME` — would couple the suite to the developer's REAL `~/.claude/fleet-roster.env` (an isolation hazard in CI / other machines). Plan:

| Test (class) | Today | After migration |
|---|---|---|
| `test_file_exists`, `…only_comments…`, `…roster_head_is_mr_radio`, `…no_user_specific_values` (Format) | open `ROSTER_PATH` (LUPIN_ROOT) | run **against a git-tracked reference/template** (e.g. `src/conf/fleet-roster.env.template` or the PIP copy) — NOT the real `~/.claude` (which isn't in the repo/CI). Format/head/no-user-value guarantees move to the template. |
| `test_clean_bash_source_exports_roster`, `…exported_to_children` (Source round-trip) | source `ROSTER_PATH` | source the **fixture/template** path in the clean bash (deterministic content), not the live user file. |
| `test_dry_run_forwards_sourced_roster` (Forward) | `_CLEAN_ENV` + LUPIN_ROOT, script reads repo file | set **`HOME=tmp_path`** with a fixture `~/.claude/fleet-roster.env`; run script (now reads `$HOME/.claude/…`); assert the PERSONA-ENV line forwards the roster. **ISOLATION UPGRADE** — no dependency on the real home. **Expand**: assert the OTHER repos forward too (`__PLAN=María`, etc.), not just `__LUPIN`. |
| `test_missing_roster_file_degrades_to_no_managers_flag` (Degrade) | LUPIN_ROOT skeleton w/o file | **`HOME=tmp_path` WITHOUT** the `.claude/fleet-roster.env` → assert no `COSA_VOICE_MANAGERS__` flag. (Mechanism shifts from LUPIN_ROOT to HOME.) |
| `test_script_syntax_ok`, `test_dry_run_exits_before_tmux` | path-agnostic | UNCHANGED. |

**Other tests:** `test_voice_persona_helpers.py:566-586` are FIXTURE-based var-format tests (`env={"COSA_VOICE_MANAGERS__LUPIN": …}`) — **path-independent, UNAFFECTED** (the var format is unchanged). `test_register_session_preservation.py:254` is a design-doc comment reference — update the comment, no code impact.

**Net test change:** re-home the path resolver (LUPIN_ROOT→HOME), introduce a fixture/template so format + round-trip tests stay hermetic, switch the script-behavior tests to a `tmp_path` HOME (an isolation improvement), and add multi-repo forward assertions. Acceptance: the suite is green against the new location with ZERO dependency on the real `~/.claude`, and proves the full multi-repo roster forwards.
