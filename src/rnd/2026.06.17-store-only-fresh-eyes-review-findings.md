## Finding 1 — Systemic false-pass: the cutover's core claim is untested on real data (HIGH)

**Context**: A known false-pass was already found — the task-list card E2E (`69f1871f`, job `ts-0e2e10c5`) passed 6/6 while the feature was broken on real data twice (invisible card → `7d169442`; render crash on array `blocked_by` → `2724b80d`). Root cause: the Playwright E2E **route-intercepts** `**/api/tasks*` with a hand-built `SEEDED_TASKS` literal and navigates straight to the section — bypassing both the real toolbar entry point and real data shapes. E2E-hardening filed as `0a0dc74c`. **Confirmed** on all three counts (`lupin/src/tests/e2e_ui/test_task_list_card.py:40-77, 105, 108-111, 134, 166-235`; the seeded `blocked_by` is a *string* `"krishna"` at line 46, so the array-crash could never surface).

**The bigger problem**: this is a *pattern*, not an isolated test. The single most load-bearing assertion of the whole cutover — **owed-work reads now come FROM THE STORE, not transcript replay** — is proven only in mock-isolated segments, never end-to-end on real data:

- **`test_stop_hook_heartbeat.py::TestRunHeartbeatStoreSource` (lines 1207-1374)** — every test patches `stop._owed_count_from_store` (lines 1236, 1253, 1264, 1286, 1309, 1328, 1344, 1358). `test_flag_on_store_owed_pokes` comments *"proves the owed source is the STORE, not replay"* but only proves that *when a mock returns `(2, True)` the hook pokes*. The HTTP call, URL params, server, and Postgres are all short-circuited.
- **No integration/E2E test** runs the real Stop hook with `owed_source_from_store=True` against a real store and asserts the count came from Postgres rows vs transcript. The cutover's "parity 4/4" is a **manual `drain --apply` run**, not an automated test.
- **`test_task_store_client.py::query_owed` (lines 201-272)** — mocks `urlopen` with an ideal `{count: N}` body. If the live server's `count_only` branch regressed (e.g. returned `len(page)` saturating at 100 — the *exact* bug `count_only` exists to prevent), this stays green.
- **`count_only` is never exercised against real Postgres**: `test_tasks_router.py:291-325` mocks the repo (`count_tasks.return_value = 273`); `test_task_repository.py:246-288` mocks the session (`query.scalar.return_value = 273`); the one real-Postgres suite `test_task_store_integration.py` uses the list path (`status="queued"`), never `count_only=true` (zero references).

**Why it matters**: the entire chain `query_owed → count_only=true → router count branch → repo.count_tasks → SQL COUNT(*)` over mirror-stamped rows has no single test exercising it on real data. A saturation or persona/project filter-mismatch bug would pass every existing test. **This is exactly the "tests green but feature broken" failure the card already demonstrated.**

**Recommendation**: an integration test (sibling to `test_task_store_integration.py`, real Postgres) that seeds owed rows via the real `POST /api/tasks` with the mirror's exact `owner_persona`/`project` stamping, then calls `_owed_count_from_store` / `query_owed` against the live server with `count_only=true` and asserts the count matches — **including a >100-row case** to actually exercise the saturation guard. This is the store-side analog of `0a0dc74c`. Owner: Mr Radio's lupin lane (filing recommended; not my build lane).

## Finding 3 — Duplicate unhardened `escapeHtml` (LOW, Tiberius's lane)

`2724b80d` hardened `escapeHtml` at `notifications.js:15257` (coerces non-strings, guards null/undefined), and added array-`blocked_by` stringification at `:9325-9327`. Both correct against the store contract (`task_store_rules.py:189-215`: `blocked_by` = non-empty list of `{kind, id}`, `id` a non-empty string).

**But there are TWO `escapeHtml` methods on the same `NotificationsUI` class** — the hardened one at `:15257` and an **OLD unhardened copy at `:8375`** (`if (!text) return ''` → `div.textContent = text`). JS resolves all ~40 call sites to the *later* definition (`:15257`), so the fix is effective everywhere **by ordering luck**. A future reorder/refactor that moves or removes the `:15257` copy would silently revert every call site. The `:8375` copy is misleading dead code.

**Recommendation**: delete the `:8375` duplicate. Minor; Tiberius's card lane.
