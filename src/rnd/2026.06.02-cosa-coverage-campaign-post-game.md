## 2. What Did Not Go Well

1. **Messaging black-holes — ONE unreliable coordination plane (FM-7/11/15/18).** Single-plane root cause, code- and disk-verified (Tiberius):
   - **Not MCP-server saturation.** `claude mcp get cosa-voice` → **Type: stdio**, Command = a per-session `.venv` Python subprocess. Every CC session spawns its *own* cosa-voice MCP subprocess over stdio — there is **no shared MCP server to saturate**. (This corrects last night's earlier two-layer "MCP saturation" framing.)
   - **The real fault: `:7999` synchronous-handler saturation.** All the per-session MCP subprocesses converge on the single shared **`:7999` FastAPI backend**, whose notify/commons handlers are **synchronous**. Under whole-fleet load (~14 sessions × frequent `commons_read`/`commons_send_to`/`notify`) those handlers block a backend worker → the MCP client's **5-second transport timeout** fires → interactive `notify()`/TTS black-holes. The observed *"Server did not respond within 5 seconds"* is that client-side budget expiring while `:7999` is blocked.
   - **Three missing properties on the one plane**: no **delivery guarantee** (synchronous + fixed-short-timeout + no retry/ack/idempotency → a *transient* `:7999` slowness = a *permanently-lost* notification); no **load-isolation** (notify/TTS rides the same `:7999` the fleet loads); no **pull-able fallback** (AFK → `user_not_available` → nothing queued for the user to read on return → FM-18).
   - This is THE headline failure; FM-7/11/15/18 are four masks on one fault. Fix-space is async/non-blocking `:7999` handlers + load-isolation + a durable, pull-able inbox — **nothing to do with the MCP transport itself**.
   - **Live specimen — the failure caught itself this morning (2026-06-02, mid-retro):** Tiberius's sanctioned `cd /tmp && claude mcp get cosa-voice` verification (the prescribed MCP check) surfaced to Rick as a **text-less default-error TTS via the Sam persona** — a dropped/empty payload with no body. That is a textbook P0 black-hole instance: the notify path failed and produced a *silent, content-free* surfacing instead of either delivering or durably queuing. The cleanest possible demonstration for Rick — the very failure this section documents, reproduced live while we wrote it up.
2. **Manufactured user-gate (FM-19) — completion-discipline gap.** In-scope, mandated work (the 100% target on websocket_manager) was reclassified as a "continue vs ramp" *user decision* because it was large and late. The 100%-full-stop mandate had already authorized completion — there was no decision to escalate. An invented checkpoint parked Rick behind it (amplified by the FM-18 notify-bounce hiding the progress).
3. **Persona-pool exhaustion / harvest-discipline failure.** Generic `extra-N` personas (🪨 "extra 1..5") appeared — a pool-exhaustion alarm that wasn't acted on. Harvest-on-unproductive was deferred and sessions piled up; the reap itself was blocked (dismiss-all permission-denied → tmux-kill workaround orphaned listeners). Self-critique on record: the MCP-restart should have been escalated far earlier instead of letting sessions accumulate.
4. **Notify-on-AFK invisibility (FM-18).** Status notifies silently bounced on `user_not_available` → ~4h of real work read as idle. Re-surface-on-return was only partially mitigated (a manual catch-up digest on Rick's return, not a pull-able status the user/manager can read on demand).
5. **(Carry-forward) TTS-brevity decay** late in the marathon — spoken payloads drifted toward markdown-dumps; refined `tts_brevity` memory captures the end-of-session decay.

## 3. Proposed Improvements (descending priority)

The headline four are what Rick asked for — a clean descending list. A lower tier of granular infra/discipline items follows so nothing is lost.

### Headline (descending)

**P0 — Reliable coordination plane** *(fixes FM-7/11/15/18 together — THE big one, Rick-named)*
- Make the `:7999` notify/commons handlers **async / non-blocking** so fleet volume can't starve interactive calls.
- **Durable queue + retry + ack/idempotency** for `notify` + commons-push (kills the silent drops).
- **Load-isolate** notify/TTS off the fleet-loaded `:7999` (separate endpoint/process).
- **Pull-able inbox / re-surface-on-AFK-return** (named deliverable — this is the FM-18 fix; folded here rather than ranked separately, since it is the same fix-family).
- Fleet concurrency-cap / backpressure on `:7999`.
- **Owners**: Tiberius → Lupin-infra root-cause + fix. María → framework §13 codification.

**P1 — Completion-discipline as a first-class principle** *(FM-19 + decision-class taxonomy; Rick-named §13-A)*
- **MANDATED in-scope work** (reach the stated target) → **NEVER user-gated**. Difficulty / lateness / size are *not* gate-triggers.
- **User-gate ONLY**: (1) outward-facing / destructive / irreversible acts (the remote push); (2) a real prod-behavior change needing design judgment; (3) a genuine requirement ambiguity (unspecified — not merely hard); (4) scope expansion beyond the mandate.
- **Valve trips ONLY** on a real prod bug (→ tripwire) or true ambiguity (→ ask) — never anticipated complexity.
- Named anti-pattern: *"escalating a non-decision wastes the user's scarce time"* — sibling of *never-make-the-user-a-tester*.
- **Owner**: María → fold into framework §5; file FM-19 with this mitigation; Tiberius cross-links from the campaign retro doc.

> **Near-tie flag (honest-steward):** P1 and P2 are a near-tied, both-Rick-named pair. Harvest (P2) was Rick's **most-repeated** directive (named multiple times), while completion-discipline (P1) is the one Rick asked us to *structurally remove* ("how did I become a gate that kept you from finishing assigned work?"). We default to P1=completion / P2=harvest; **Rick may swap them in one word.**

**P2 — Harvest-discipline MANDATE** *(Rick-named §13-C, his most-repeated)*
- `extra-N` appearance = **persona-pool-exhaustion ALARM** → harvest-on-unproductive becomes a MANDATE, not a deferral; escalate the MCP-restart/reap early rather than letting sessions pile.
- Honest-shutdown / reaping: auto-reap-on-completion, stall-threshold sweep, liveness sweep. Fix the reap path itself — `dismiss_sessions` list-arg stringify bug + SIGKILL-by-exact-PID for the orphaned `ppid=1` listeners (root-caused + resolved last night; see §1 WENT-WELL #6).
- **Owners**: Tiberius → Lupin `02-cold-start-runbook` mandate + reap fix. María → framework §13 carry.

**P3 — TTS-brevity: HOOK-ENFORCED hard cap** *(elevated 2026-06-02 — Rick re-raised "wall of text" the morning after the identical correction → discipline-by-memory has demonstrably failed)*
- Spoken `notify(message=...)` = ONE headline sentence + AT MOST TWO takeaways; every number/name/path/list/rationale → `abstract` + doc-links. Terminal close ≤ ~5 lines.
- **Systematic mechanism (the real fix):** a notify-wrapper / stop-hook that hard-caps the spoken `message` (rejects or truncates > ~40 words / > 3 sentences) so brevity is not dependent on per-turn willpower. Self-enforcement decays; automate it.
