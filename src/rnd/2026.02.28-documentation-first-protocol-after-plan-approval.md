# Plan: Documentation-First Protocol After Plan Approval

## Context

When Claude Code exits plan mode and begins implementation, it immediately jumps to writing code — even when the plan specifies documentation artifacts (planning docs, tracking docs, architecture notes) that should be created in `src/rnd/` **first**. The user always specifies during the planning phase which documents they want generated, but Claude ignores this sequencing and starts coding.

This isn't specific to any one workflow (like `/p-is-p-02-documentation`). It happens with Claude Code's native plan mode UI toggle. The fix needs to be a global behavioral directive that applies to ALL projects after ANY plan is approved.

**Intended behavior**: Plan approved → create ONLY the `.md` documentation artifacts specified in the plan → serialize to `src/rnd/` → `ask_yes_no()` confirmation → ONLY then begin code implementation if user says yes.

**Content to add**:

```markdown
### DOCUMENTATION-FIRST PROTOCOL

**MANDATE**: After plan mode produces a plan that specifies documentation artifacts, those artifacts MUST be created BEFORE any code is written.

**The Rule**: When a plan mentions documents to create (planning docs, tracking docs, architecture notes, research syntheses, implementation guides), treat documentation creation as **Phase 0** of implementation — a mandatory prerequisite that must complete before any code files are touched.

**Sequence**:
1. Plan is approved → exit plan mode
2. **FIRST**: Create/serialize ALL documentation artifacts specified in the plan (`.md` files in `src/rnd/` or other locations the plan specifies)
3. **THEN**: Confirm via cosa-voice before writing any code:
   ```python
   ask_yes_no(
       "Documentation complete. Ready to begin code implementation?",
       default="no",
       priority="high",
       abstract="**Documents created**:\n- [list each file]\n\nSay YES to begin coding, NO to revise documentation first."
   )
   ```
4. **ONLY IF user says yes**: Begin writing code

**ABSOLUTE PROHIBITIONS** (during documentation phase):

| Action | Why It's Forbidden |
|--------|-------------------|
| **NEVER** write code files before documentation is complete | Documentation artifacts are the user's planning checkpoint |
| **NEVER** assume documentation step can be skipped | If the plan mentions documents, they are required |
| **NEVER** combine documentation and code in the same "step" | They are distinct phases with a gate between them |

**FILE EXTENSION RULE** (during documentation phase):
- `.md` files → ALLOWED (documentation artifact)
- All other extensions → PROHIBITED until user confirms via `ask_yes_no()`

**When this does NOT apply**:
- Plans that specify NO documentation artifacts (pure code-only plans)
- Trivial plans (<1KB) that don't mention any documents to create
- When the user explicitly says "skip docs and start coding"
```

### 2. Plan Serialization Skill (`~/.claude/skills/plan-serialization/SKILL.md`)

**What**: Add a new section reinforcing the documentation-first protocol.

**Where**: Insert after the `## Integration Points` section (after line 92), before `## Anti-Patterns` (line 95).

**Content to add**:

```markdown
## Documentation-First Protocol

**MANDATE**: Plan serialization is part of a broader documentation-first protocol. When a plan specifies documentation artifacts to create:

1. **Serialize the plan itself** to `src/rnd/` (this skill's primary function)
2. **Create any additional documents** the plan specifies (tracking docs, architecture notes, etc.)
3. **Confirm with user** via `ask_yes_no()` before writing any code
4. **Only then** begin code implementation

This means plan serialization is NOT the last step — it's the FIRST step of a documentation phase that must complete before coding begins.

### Trigger Recognition

When reading an approved plan, look for documentation signals:
- "Create a planning document..."
- "Generate tracking documentation..."
- "Document the architecture in..."
- "Write a research synthesis..."
- Any mention of `.md` files to be created in `src/rnd/`

If ANY of these signals are present, the documentation-first protocol applies.
```
