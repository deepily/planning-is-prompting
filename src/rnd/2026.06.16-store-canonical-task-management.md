**Date**: 2026-06-16 (rev 2026-06-17) · **Author**: María 🌸 (Workflow Steward) · **Status**: ✅ APPROVED (Rick GO broadcast `42c3e814`) + cascade APPROVE-WITH-REVISIONS (unanimous 3/3) — build underway. See § Cascade Review Outcome.

## Context — why

The founding goal is a **token-efficient** way to track status + liveness so the fleet can **drive lazy / stuck / missing-something workers to completion** — without 17,000 coffee-break belly-scratch stalls. Every problem we hit tonight traces to **one root**: the architecture leans on the **native Claude Code harness task list** and tries to keep it and the unified store in sync.

That native list is the wrong substrate for fleet liveness:
- It is **vocabulary-poor** (`pending/in_progress/completed` — no `blocked`, `blocked_by`, `next_chase`, owner, accountable, receipts).
- It is **only model-writable** (only `TaskCreate`/`TaskUpdate` fill it; no external push) and **only transcript-reconstructable** (no session file).
- Keeping it in sync with the store spawned the **mirror** and its entire bug class (`9bf1dc4a`, `9b23d5bc`, `82e4eaf0`).
- Any "always-current" sync of it into context is a **per-turn token cost** — a non-starter.

**Decision (Rick, 2026-06-16): jettison the native harness task list from the architecture.** Track owed work **solely** in our own unified store. The heartbeat poke and the arbiter **both query that one store**; the human sees a **dedicated UI card** rendered from the store (exactly like the fleet-status card today). This deletes the mirror, the vocabulary mismatch, the transcript replay, the echo gotcha, and the per-turn injection — all at once — and makes divergence **impossible** because there is only one source.

## The architecture (one source, three readers)

```
                       ┌──────────────────────────┐
   writes (task_*)  →  │   UNIFIED TASK-STORE      │  ← the ONLY source of truth
                       │   (:7999 /api/tasks)      │
                       └─────────┬───────┬─────────┘
            ┌────────────────────┘       └────────────────────┐
   (1) Heartbeat poke            (2) Arbiter (:8001)     (3) Human UI card
   Stop-hook COUNT query         same COUNT/owed query    multiplexer card, rendered
   "do I owe work? how many?"    cross-fleet liveness      from task_query (like
   → poke if owed + no hold      → drive/escalate          fleet-status today)
```

- **(1) Liveness / self-poke — token-cheap.** The Stop hook makes one small `task_query(owner_persona=self, status in {queued,in_progress})` at **stop-time** and pokes if the count is > 0 and no fresh hold. It needs the **count**, not the list; **nothing is injected into context**. The poke says *"you owe N items,"* not the items. Cost ≈ one cheap query per *stop*, not per *turn*.
- **(2) Arbiter — same source.** The `:8001` arbiter queries the **same** store for cross-fleet liveness/owed-work. Because (1) and (2) read one source, **they cannot disagree** — bug `82e4eaf0` (two sources of truth) is eliminated by construction, not patched.
- **(3) Human display — a dedicated UI card.** A task-list card in the cosa-voice / multiplexer UI, rendered from `task_query` via an API, **exactly like the fleet-status card is rendered now.** This replaces the native Claude Code task widget. It can show full fidelity (blocked, blocked_by, next_chase, owner, accountable) because it reads the store directly.

**The native Claude Code harness task list is NOT used** for tracking owed work. No `TaskCreate`/`TaskUpdate` for fleet work; no mirror; no hydration of a native widget.

## Part 1 — The doctrine mandate (MUST / MUST NOT / NEVER / ALWAYS)

> ### MANDATE — the unified task-store is the ONLY task list
> 1. You **MUST** write every unit of owed work — your tasks, work you assign, decisions, bugs, gates — to the **unified task-store** (`task_*` verbs / `:7999 /api/tasks`). It is the one and only system of record.
> 2. You **MUST NOT** use the native Claude Code harness task list (`TaskCreate`/`TaskUpdate`) to track owed work. It is **jettisoned** from the fleet architecture — not a mirror source, not a fallback, not a parallel ledger.
> 3. **NEVER** let a unit of owed work live only in your context or your head. If it is not a store row, it is invisible to the poke, the arbiter, and the fleet — a discipline bug.
> 4. The store **ALWAYS** wins and is the single source: the heartbeat poke **and** the arbiter both read it, so they cannot diverge. To *see* your list, **query the store on demand** — do not keep a second copy.
> 5. You **MUST** keep each item's status current (`queued → in_progress → blocked → done`), **ALWAYS** with evidence: `→blocked` carries a typed `blocked_by` **and** a `next_chase_ts`; `→done` carries a receipt. No receipt → not done.
> 6. The human-visible task list is a **dedicated UI card rendered from the store** (like fleet-status) — **NEVER** the native task widget.

## Tradeoffs addressed

- **The model loses its native in-context working list.** With the native list jettisoned, the model no longer has the auto-rendered TodoWrite widget for self-organization. Resolution: the model **queries the store on demand** when it needs to see its list (session-start, or when picking the next item) — token-cheap because it's *on demand*, not per-turn, and it's the same `task_query` the poke uses. Day-to-day, the model knows what it's working on without re-reading the full list.
- **Store as a runtime dependency of the Stop hook.** The hook must reach `:7999` at stop-time. It already makes outbound calls (arbiter, commons); the store query is the same shape. The store-down fail-safe (build step 2) prevents spurious pokes when it can't.
- **No native widget for Rick.** Intentional — replaced by the richer, full-fidelity UI card (build step 4), which can show `blocked`/`next_chase`/owner that the native widget never could.
