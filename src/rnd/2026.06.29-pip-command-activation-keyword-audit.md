## The core finding

A `.claude/commands/*.md` file is reachable **only** by typing its slash command. A `.claude/skills/<name>/SKILL.md` with a trigger-rich `description:` frontmatter is **semantically matched** — it auto-activates when the user's words match. Among PIP surfaces, **only four carry a real trigger catalog**; everything else is explicit-only.

## Collision map (semantic-trigger overlaps to watch)

| Phrase | Could match | Status |

| "review this plan" | `plan-review` ↔ `plan-review-cascaded` ↔ harness `code-review` | Safe today (PIP review surfaces are title-only, no auto-fire). **Risk if either becomes a skill** — would need single-vs-cascaded disambiguation. |

| "walk me through / decide" | `guided-decision-walkthrough` | Owned. |
| "spin up team" | `spin-up-swe-team` | Owned. |
| "wrap up / end the session" | `plan-session-end` (title-only — won't auto-fire) | `plan-post-game` deliberately avoids "wrap up" so the two never blur if session-end later becomes a skill. |

## Recommendations (for Rick's call — not yet actioned)

1. **Candidates worth promoting to full skills** (so natural language fires them): `plan-session-start` / `plan-session-end` ("let's start/wrap the session") and `plan-review` ("review this plan"). High everyday-use, currently only reachable by remembering the slash command.
2. **Before promoting `plan-review` / `plan-review-cascaded`**: define disambiguating triggers (single-reviewer vs cascaded) so "review this plan" routes deterministically. The post-game carve-out already proves the pattern — keep each surface's trigger set mutually exclusive.
3. **Keep the rest explicit-only.** Low-frequency / destructive / setup commands (backup, wizard, branch-PR, loc-delta) are *better* as explicit-only — you don't want "back up" auto-firing a write.
